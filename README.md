# bayesian-portfolio

Bayesian portfolio optimization implementing the Black-Litterman model from scratch, extended with a conjugate Normal-Wishart prior on the return distribution. Closed-form efficient frontiers via Lagrangian KKT systems. No sklearn, no cvxpy, no probabilistic programming libraries — pure NumPy.

## Mathematical Overview

### Normal-Wishart Conjugate Prior

The joint prior on (μ, Σ) is:

$$p(\mu, \Sigma) = \mathcal{N}(\mu \mid \mu_0, \Sigma/\kappa_0) \cdot \mathcal{W}^{-1}(\Sigma \mid \nu_0, \Psi_0)$$

After observing T returns, the posterior updates in closed form:

$$\kappa_n = \kappa_0 + T, \quad \mu_n = \frac{\kappa_0 \mu_0 + T\bar{x}}{\kappa_n}, \quad \Psi_n = \Psi_0 + S + \frac{\kappa_0 T}{\kappa_n}(\bar{x} - \mu_0)(\bar{x} - \mu_0)^\top$$

### Black-Litterman Master Formula

Prior: μ ~ N(Π, τΣ) where Π = δΣw_mkt. View likelihood: Pμ = Q + ε, ε ~ N(0, Ω).

$$\mu_{\text{BL}} = M^{-1}\!\left[(\tau\Sigma)^{-1}\Pi + P^\top \Omega^{-1} Q\right], \quad M = (\tau\Sigma)^{-1} + P^\top \Omega^{-1} P$$

All systems solved via `np.linalg.solve` — `np.linalg.inv` is never called.

### Efficient Frontier (Closed-Form KKT)

Min-variance: `w* = Sigma^{-1} 1 / (1^T Sigma^{-1} 1)` via `solve(Sigma, ones)`.  
Max-Sharpe: `w* = Sigma^{-1}(μ - rf) / sum(...)` via `solve(Sigma, excess)`.  
Target return: bordered Hessian (n+2)×(n+2) system solved by `np.linalg.solve`.

## Quick Start

```python
import numpy as np
from bayesian_portfolio import (
    NormalWishartPrior, NormalWishartPosterior,
    BlackLitterman,
    SampleCovariance, LedoitWolfCovariance,
    EfficientFrontier,
)

# 1. Bayesian covariance estimation
returns = np.random.randn(500, 10) * 0.01
prior = NormalWishartPrior(mu_0=np.zeros(10), kappa_0=1.0, nu_0=12.0, Psi_0=np.eye(10))
post = NormalWishartPosterior.update(prior, returns)
mu, Sigma = post.posterior_mean(), post.posterior_covariance()

# 2. Black-Litterman with one view
w_mkt = np.ones(10) / 10
bl = BlackLitterman(Sigma=Sigma, w_mkt=w_mkt, delta=2.5, tau=0.05)
P = np.zeros((1, 10)); P[0, 0] = 1.0; P[0, 1] = -1.0
Q = np.array([0.05 / 252])
Omega = bl.default_omega(P)
mu_bl, Sigma_bl = bl.posterior(P, Q, Omega)

# 3. Efficient frontier
ef = EfficientFrontier(mu=mu_bl, Sigma=Sigma_bl)
w_mv = ef.min_variance()
w_ms = ef.max_sharpe()
frontier = ef.compute_frontier(n_points=200)
print(frontier[['portfolio_return', 'portfolio_std', 'sharpe_ratio']].head())
```

The test suite covers:
- Conjugate posterior updates (mean convergence under prior strength extremes, positive definiteness).
- Black-Litterman formulation (no-view equilibrium recovery, multi-asset blending behavior, implied return round-trips).
- Covariance estimators (shrinkage bounds, dimensional validation).
- Closed-form KKT frontiers (weight constraints, global minimum variance, maximum Sharpe ratio, target return portfolios).

## Covariance Estimators

| Estimator | Method | Key Parameter |
|-----------|--------|---------------|
| `SampleCovariance` | Biased MLE (÷T) | — |
| `LedoitWolfCovariance` | OAS shrinkage toward μI | `shrinkage_intensity_` ∈ [0,1] |
| `NormalWishartCovariance` | Posterior mean of Inverse-Wishart | `prior_strength_ratio_` |

All share the interface: `fit(returns)` → `covariance_`, `correlation_`, `condition_number_`.

## Design Decisions

- **No `np.linalg.inv`** — all linear systems solved via `np.linalg.solve`
- **No external optimization/ML packages** — no sklearn, cvxpy, scipy.optimize, or PyMC/Stan; conjugate posteriors and KKT systems solved in pure NumPy
- **Deterministic seeds** — seed set at 42 to ensure reproducible simulations
- **No data leakage** — strict date ordering asserted in walk-forward evaluation

## Limitations

1. **Gaussianity**: fat tails and volatility clustering violate the i.i.d. Normal assumption.
2. **Stationarity**: correlations and expected returns shift across macro regimes.
3. **Sensitivity to parameters**: the Black-Litterman prior is highly sensitive to the risk aversion parameter δ and market portfolio weights w_mkt.
4. **Garbage-in, garbage-out**: Bayesian methods do not compensate for low-quality or highly biased view returns Q.
5. **High-dimensional scaling**: requires T ≫ n for reliable covariance estimation; factor models are preferred for n > 100.

See Section 8 of `notebooks/analysis.ipynb` for a detailed discussion.
